Permintaan di atas menunjukkan contoh serangan di mana penyerang mencoba mengunggah file berbahaya ke server. File ini tampak seperti gambar (dengan nama "sample.php%00.jpg") namun sebenarnya berisi kode PHP (<?php phpinfo(); ?>) yang bisa dieksekusi di server jika tidak ada pemeriksaan keamanan yang ketat. Saat server menerima file ini tanpa validasi yang benar, kode PHP di dalam file tersebut dapat dijalankan, memungkinkan penyerang mengakses informasi sistem atau menjalankan perintah tidak sah. Untuk mencegah serangan seperti ini, server harus memeriksa isi file, bukan hanya ekstensi atau nama file, untuk memastikan bahwa hanya file yang benar-benar berformat gambar yang diizinkan, dan menyimpan file di lokasi yang aman agar tidak bisa dieksekusi.
